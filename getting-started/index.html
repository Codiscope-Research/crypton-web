Getting Started

Crypton is not yet ready for a production environment.

The following will guide you through setting up a Crypton server for development. This guide assumes you are on Ubuntu, though this is not required.

Dependencies

The Crypton server depends on PostgreSQL and Node.js v0.8.x. 

# install postgres
sudo apt-get install postgresql

# install git
sudo apt-get install git-core

# install node.js dependencies
sudo apt-get install g++ curl libssl-dev apache2-utils

# install node.js
git clone https://github.com/joyent/node
cd node
./configure
make
sudo make install

# get crypton
git clone https://github.com/SpiderOak/crypton.git

# setup the database
cd crypton/integration_tests
make setup-test-environment

Running the server

cd crypton/server
npm link
which crypton # should be ./server.js
NODE_ENV=test crypton

To clean the database (remove all data):

cd crypton/integration_tests
make clean-test-db

Client compilation

To generate the un- and minified versions of the client code:

cd crypton/client
./compile.sh --once
# crypton.js and crypton.min.js should now be in crypton/client/dist
# if --once is left off, compile.sh recompile once a change in the client directory is detected
